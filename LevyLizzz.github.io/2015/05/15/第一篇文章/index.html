Suck

今天建站9点钟弄到现在都没有午休。呀！估计查了100个页面，说下自己的心得。我就是想有个地方放放东西，最重要的啦还是以后有个地方测试自己的小Demo。然后看到Hexo这个模块Style挺喜欢，然后谢谢萌得以为是妹子的地瓜菌给我指导了很多。

安装node.js

nodejs.org直接下载msi安装全部确认，有疑问百度一下。

安装hexo

git bash里输入

hexo的一些命令先贴了：

Github配置

deploy这个指令没有push上去，所以我是手动部署的，肯定不是最好的方法，但是我达到了我的目的，我会优化的。
Github注册一个月了都不知道放什么东西也没看，今天操作一波git指令简直爽疯

New Repository

Repository Name一定要写成[Your UserName].github.io，这样这个仓库的项目才会有publish到GitHub Pages上。地址在repository setting里面查看。
初始化hexo/.deploy_git文件，还有最好点击自动生成README.md。

Publish Github Pages

进入…/.deploy_git这个目录右键Git bash：

Repository Settings里面GitHub Pages选项会记录 Your site is published at http:// [Your UserName].github.io，打开命令提示符（运行CMD）ping一下这个地址的IP，然后把自己域名 的DNSpod解析到这个IP就。
之后在你的项目仓库下创建一个名为CNAME的文件，写入你自定义的域名，应该页面就publish上去了。

GitCafe Pages(国内这个比较快呢)

在GitCafe里面创建一个和你用户名一样的项目，然后把本地项目push上去，具体请看Gitcafe Pages，在项目设置的Pages服务中添加自己的域名就OK了。

$ Hexo deploy

hexo deploy可以在项目根目录下的_config.yml中配置两个push地址
deploy:
type: git
repo:
github: git@github.com:[your username]/[your username].github.io.git,master
gitcafe: git@gitcafe.com:[your username]]/[your username].git,gitcafe-pages
但是要统一配置username,usermail以及SSH公匙。

壮士请留步！你肯定还会回来看的。
我今天碰了一万个

error：

remote origin already exists.

托管代码已经存在，所以要把托管的内容删掉：

Could not read from remote repository.Please make sure you have the correct access rights.and the repository exists.

在终端输入。

如果执行成功。返回
Generating public/private rsa key pair.
Enter file in which to save the key (/home/forwhat.cn/.ssh/id_rsa):
在这里就是设置存储地址了.反正我是直接按的回车
然后还会返回
Enter passphrase (empty for no passphrase):
再次直接回车。
Enter same passphrase again:
再次回车。
Your identification has been saved in /home/forwhat.cn/.ssh/id_rsa.
Your public key has been saved in /home/forwhat.cn/.ssh/id_rsa.pub.
The key fingerprint is:
这里有一串数。我把他屏蔽了。
The key’s randomart image is:
+—[ RSA 2048]——+
| |
| |
| |
| o |
| + . S |
| . = . o |
| o + +o.o |
|E o . o.=+. |
|.+ ==+ooo. |
+————————-+
这样SSH key就生成了。直接cat一下就好了。
000000:~/a$ cat /home/forwhat.cn/.ssh/id_rsa.pub
ssh-rsa AAAAB3NzaC1yc2EAAAADAQ(我屏蔽了)

把显示出来的直接添加到github账户设置里边的SSH keys。
好了网上的流程告诉我这样可以解决了，但是我又FATAL:

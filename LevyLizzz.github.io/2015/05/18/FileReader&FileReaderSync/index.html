FileReader：异步读取一个File或Blob
FileReader定义读取File或Blob的异步API。可通过下面步骤读取一个文件：

  • 通过FileReader()构造函数创建一个FileReader。
  • 定义需要的事件处理程序。
  • 将File或Blog对象传入4个读取方法中的一个。
  • 触发onload处理程序时，文件内容将可以通过result属性访问。或者，如果触发onerror处理程序，error属性将指向一个提供更多信息的FileError对象。
  • 读取完成是，可以重用FileReader对象或根据需要丢弃它并创建一个新的。

FileReaderSync：同步读取一个File或Blob
FileReaderSync是FileReader API的一个同步版本，仅在Worker线程中可用。同步API比异步更好用：只需要简单地创建一个FileReaderSync()对象，然后调用它的一个读取方法，改方法将返回对应的File或Blob的内容或者抛出一个FileError对象。

——《JavaScript权威指南》

FileReader

方法

方法名 描述
abort() 中止一个读取任务
readAsArrayBuffer(Blob blob) 异步读取blob的字节内容，结果为一个ArrayBuffer对象，可通过result属性访问
readAsBinaryString(Blob blob) 异步读取blob的字节内容，将它们编码为JavaScript二进制字符串，并将结果赋值给result属性。
readAsDataURL(Blob blob) 异步读取blob的字节，将它们(以及该Blog的类型)编码为data://URL,并将result属性设置为返回字符串。
readAsText(Blob blob,[Sting encoding]) 异步读取blob的字节内容，并使用指定的encoding将它解码为一个Unicode文本字符串，然后将result属性设置为解码后的字符串。如果未指定encoding,将默认使用UTF-8。

事件

事件 描述
onbort 当读取任务abort()方法中止时触发。
onerror 发生某种错误时触发。FileReader的error属性指代一个有错误代码的FileError对象。
onload 当File或Blog成功读取时触发。FileReader的result属性值为File或Blob的内容，形式取决于调用的是哪个读取方法。
onloadend 每次调用FileReader的读取方法最后都会产生一个load事件、一个error事件或一个abort事件。在这些事件之后FileReader也会触发一个loadend事件，这样有些脚本就可以只监听一个事件而不是所有三个事件。
onloadstart 在调用一个read方法之后但是读取数据之前触发。
onprogress 当正在读取File或Blob数据时触发，大约每秒20次。ProgressEvent对象指出已经读取了多少字节，FileReader的result属性可能包含这些字节的表示形式。

方法

方法名 描述
ArrayBuffer readAsArrayBuffer(Blob blob) 读取blob的自己内容并以ArrayBuffer的形式返回。
string readAsBinaryString(Blob blob) 读取blob的字节内容，并编码JavaScript二进制字符串，然后返回这个二进制字符串。
string readAsDataURL(Blob blob) 读取blob的字节内容并将内容与blob的type属性编码为一个data://URL，返回这个URL。
string reaAsText(Blob blob,[string encoding]) 读取blob的字节内容兵使用指定的encoding（如果没有指定编码则使用UTF-8或UTF16）解码为文本，并返回这个文本。
